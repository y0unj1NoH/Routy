# 백엔드 테이블 명세 (최종 MVP)

루트 `README.md` 최종 설계 기준으로 정리한 DB 구조입니다.

## ERD

```text
auth.users
    ├── place_lists
    │     └── place_list_items ─── places
    └── schedules
          └── schedule_days
                └── schedule_stops ─── places
```

## 1) places (공용 장소 원본)

- `id uuid PK`
- `google_place_id text UNIQUE`
- `name text`
- `formatted_address text`
- `lat/lng double precision`
- `rating numeric`
- `user_rating_count integer`
- `price_level smallint`
- `types_raw jsonb`
- `category text`
- `opening_hours jsonb`
- `photos jsonb`
- `reviews jsonb`
- `phone text`
- `website text`
- `google_maps_url text`
- `created_at`, `updated_at`

RLS: authenticated 사용자 읽기/쓰기 허용 (공용 캐시 모델).

## 2) place_lists (유저 리스트 헤더)

- `id uuid PK`
- `user_id uuid FK -> auth.users(id)`
- `name text`
- `city text`
- `description text`
- `created_at`, `updated_at`

RLS: `user_id = auth.uid()`

## 3) place_list_items (리스트 아이템)

- `id uuid PK`
- `list_id uuid FK -> place_lists(id)`
- `place_id uuid FK -> places(id)`
- `note text`
- `priority boolean` (`MUSTVISIT` 원천)
- `created_at`
- `UNIQUE(list_id, place_id)`

RLS: 소유한 `place_lists` 기준 접근.

## 4) schedules

- `id uuid PK`
- `user_id uuid FK -> auth.users(id)`
- `title text`
- `start_date date`
- `end_date date`
- `day_count integer`
- `place_list_id uuid FK -> place_lists(id)`
- `stay_place_id uuid FK -> places(id)`
- `companions text`
- `pace text`
- `themes jsonb`
- `generation_input jsonb`
- `generation_version text`
- `is_manual_modified boolean`
- `created_at`, `updated_at`

RLS: `user_id = auth.uid()`

## 5) schedule_days

- `id uuid PK`
- `schedule_id uuid FK -> schedules(id)`
- `day_number integer`
- `date date`
- `created_at`
- `UNIQUE(schedule_id, day_number)`

RLS: 부모 schedule 소유권 기반.

## 6) schedule_stops

- `id uuid PK`
- `schedule_day_id uuid FK -> schedule_days(id)`
- `stop_order integer`
- `place_id uuid FK -> places(id)`
- `time time`
- `label text`
- `badges jsonb`
- `note text`
- `reason text`
- `transport_to_next jsonb`
- `is_user_modified boolean`
- `created_at`
- `UNIQUE(schedule_day_id, stop_order)`

RLS: `schedule_day -> schedule -> user_id` 체인으로 소유권 검증.
